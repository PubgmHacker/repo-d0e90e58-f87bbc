    19‚Üí#### C1. WebSocket `isConnected` is never set to `true` ‚Äî entire realtime send path is dead
    20‚Üí- **File:** `Networking/WebSocketClient.swift:408-424`
    21‚Üí- **What's wrong:**
    22‚Üí  ```swift
    23‚Üí  private func notifyConnectedIfNeeded() {
    24‚Üí      guard !isConnected else { return }
    25‚Üí      isConnected = true            // ‚Üê only place isConnected is set to true
    26‚Üí      ...
    27‚Üí      delegate?.webSocketDidConnect(self)
    28‚Üí  }
    29‚Üí  ```
    30‚Üí  `notifyConnectedIfNeeded()` is declared but **never called anywhere** in
    31‚Üí  the codebase (verified via grep). `URLSessionWebSocketTask` has no "did
    32‚Üí  open" callback, so the comment claims to "infer open state from the
    33‚Üí  first successful receive cycle / probe openness with a no-op send", but
    34‚Üí  neither is implemented.
    35‚Üí- **Impact:** `send(_:)` at line 221 checks `if isConnected` (always
    36‚Üí  `false`) ‚Üí every outgoing message is enqueued to `pendingMessages`
    37‚Üí  and **never flushed** (flush only happens inside
    38‚Üí  `notifyConnectedIfNeeded`). `sendHeartbeat` short-circuits at line 337.
    39‚Üí  `delegate?.webSocketDidConnect(self)` is never invoked, so
    40‚Üí  `RoomViewModel.connectionStatus` stays `.connecting` forever and the
    41‚Üí  sync engine never starts. Chat, reactions, play/pause/seek,
    42‚Üí  signaling ‚Äî **none of it reaches the server**.
    43‚Üí- **Fix:** Use `urlSession.webSocketTask(with:)`'s implicit open state by
    44‚Üí  probing with a no-op receive right after `task.resume()`:
    45‚Üí
    46‚Üí  ```swift
    47‚Üí  private func connectInternal() {
    48‚Üí      ...
    49‚Üí      task.resume()
    50‚Üí      socket = task
    51‚Üí      // Probe open state by issuing an immediate receive; the first
    52‚Üí      // successful (or first failure) callback tells us the socket is live.
    53‚Üí      receiveMessage()
    54‚Üí      // Also schedule a one-shot open-check
    55‚Üí      Task { @MainActor [weak self] in
    56‚Üí          try? await Task.sleep(nanoseconds: 250_000_000)
    57‚Üí          self?.notifyConnectedIfNeeded()
    58‚Üí      }
    59‚Üí  }
    60‚Üí  ```
    61‚Üí  Or, more cleanly, open the socket via a `withCheckedThrowingContinuation`
    62‚Üí  wrapper that resumes on the first receive callback.

    93‚Üí#### C3. `AuthService.getFreshToken()` never actually refreshes
    94‚Üí- **File:** `Services/AuthService.swift:118-124`
    95‚Üí- **What's wrong:**
    96‚Üí  ```swift
    97‚Üí  func getFreshToken() async -> String? {
    98‚Üí      let now = Date().timeIntervalSince1970
    99‚Üí      if authToken == nil || now >= tokenExpiry - 300 {
   100‚Üí          return authToken        // ‚Üê same value as the else branch
   101‚Üí      }
   102‚Üí      return authToken            // ‚Üê identical

   104‚Üí  ```

   106‚Üí  There is no refresh-token call to `/api/auth/refresh` or similar.

   109‚Üí  or any in-app recovery path. The `bridgeAuthToken()` call in
   110‚Üí  `RaveCloneApp.init` then propagates an expired token to WebSocketClient
   111‚Üí  and MediaService.
   112‚Üí- **Fix:**
   113‚Üí  ```swift
   114‚Üí  func getFreshToken() async -> String? {
   115‚Üí      guard let token = authToken else { return nil }
   116‚Üí      let now = Date().timeIntervalSince1970
   117‚Üí      // Refresh if within 5 min of expiry (or past it)
   118‚Üí      if now >= tokenExpiry - 300 {
   119‚Üí          return await refreshJWT() ?? token
   120‚Üí      }
   121‚Üí      return token
   122‚Üí  }
   123‚Üí  private func refreshJWT() async -> String? { /* POST /auth/refresh */ }
   124‚Üí  ```
